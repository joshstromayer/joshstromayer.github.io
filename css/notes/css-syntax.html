CSS Syntax

Welcome to the CSS Syntax tutorial! In this section, you'll learn about the building blocks of CSS and how to write proper styles for your web pages. Understanding the syntax is the foundation of mastering CSS and creating beautiful designs.

The Basic Structure of CSS

CSS rules are made up of two main parts: a selector and a declaration block. Together, they define which HTML elements should be styled and how they should look.

selector {
  property: value;
}

Here's a breakdown of the syntax:

  • Selector: Specifies the HTML element(s) you want to style (e.g., h1, p, or .class).
  • Declaration Block: Contains one or more declarations, each specifying a property and a value.
  • Property: The style you want to apply, such as color or font-size.
  • Value: Specifies the setting for the property, such as red or 16px.

Example of a CSS Rule

Let's take a look at a simple example that changes the color and font size of all h1 elements:

h1 {
  color: blue;
  font-size: 24px;
}

Selectors

Selectors define which elements the styles should apply to. For example:

  • Type Selector: Targets an HTML tag, like p or h1.
  • Class Selector: Targets elements with a specific class, using a dot (e.g., .classname).
  • ID Selector: Targets an element with a specific ID, using a hash symbol (e.g., #idname).

You'll learn more about selectors in the CSS Selectors tutorial.

Multiple Declarations

You can include multiple declarations within a single declaration block by separating them with semicolons. For example:

p {
  color: red;
  font-size: 18px;
  text-align: center;
}

Best Practices for Writing CSS

  • Use consistent indentation to make your CSS easier to read.
  • Add comments to explain complex styles or sections of your code.
  • Group related styles together to keep your CSS organized.
