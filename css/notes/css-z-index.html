CSS Z-Index

The z-index property in CSS controls the stacking order of elements on a webpage. Elements with a higher z-index value are displayed in front of elements with a lower value. This is particularly useful when elements overlap.

Default Stacking Order

By default, elements are stacked according to the order they appear in the document. Later elements are displayed on top of earlier ones unless z-index is used to change this behavior.

Using Z-Index

The z-index property works only on elements that have a position value other than static (e.g., relative, absolute, fixed, or sticky). Here’s the syntax:

<style>
.box1 {
  width: 100px;
  height: 100px;
  background-color: lightblue;
  position: absolute;
  top: 50px;
  left: 50px;
  z-index: 1;
}
.box2 {
  width: 100px;
  height: 100px;
  background-color: lightcoral;
  position: absolute;
  top: 80px;
  left: 80px;
  z-index: 2;
}
</style>
<div class="box1">Box 1</div>
<div class="box2">Box 2</div>

In this example, Box 2 overlaps Box 1 because it has a higher z-index value.

Negative Z-Index

You can assign a negative value to the z-index property, which places the element behind others with a z-index of 0 or greater:

<style>
.background {
  width: 200px;
  height: 200px;
  background-color: lightgray;
  position: relative;
  z-index: -1;
}
.foreground {
  width: 100px;
  height: 100px;
  background-color: lightpink;
  position: relative;
}
</style>
<div class="foreground">Foreground</div>
<div class="background">Background</div>

In this example, the background element is placed behind the foreground element due to its negative z-index.

Managing Complex Stacking

When dealing with multiple overlapping elements, the stacking order is determined by the z-index value. However, elements within the same stacking context cannot overlap based on their parent’s z-index. A new stacking context is created when:

  • An element has a z-index value other than auto and a position value other than static.
  • An element has certain CSS properties like opacity less than 1 or transform applied.
<style>
.parent {
  width: 200px;
  height: 200px;
  background-color: lightgreen;
  position: relative;
  z-index: 1;
}
.child {
  width: 100px;
  height: 100px;
  background-color: lightblue;
  position: absolute;
  top: 50px;
  left: 50px;
  z-index: 2;
}
</style>
<div class="parent">
  Parent
  <div class="child">Child</div>
</div>

In this example, the child element has a higher z-index than its parent, but its stacking is still confined within the parent’s stacking context.

Practical Example

Use the z-index property to create a layered design:

<style>
.layer1 {
  width: 150px;
  height: 150px;
  background-color: lightcoral;
  position: absolute;
  z-index: 1;
}
.layer2 {
  width: 100px;
  height: 100px;
  background-color: lightblue;
  position: absolute;
  z-index: 2;
  top: 25px;
  left: 25px;
}
.layer3 {
  width: 50px;
  height: 50px;
  background-color: lightgreen;
  position: absolute;
  z-index: 3;
  top: 50px;
  left: 50px;
}
</style>
<div class="layer1">Layer 1</div>
<div class="layer2">Layer 2</div>
<div class="layer3">Layer 3</div>

This creates a layered structure where each layer is positioned above the previous one using the z-index property.
