What Are Tags and Elements?

HTML is built using tags and elements. These are the building blocks of every webpage you see online.


Tags

A tag is used to mark the beginning and (sometimes) the end of an HTML element. Tags tell the browser how to display content.

Tags are written inside angle brackets, like this: <tagname>.

For example, the <h1> tag tells the browser to display a heading.


Elements

An element in HTML is the combination of a start tag, content, and an optional end tag.

For example:

<h1>This is a heading</h1>
<p>This is a paragraph.</p>

In this example:

  • The <h1> tag is the start tag.
  • The text This is a heading is the content.
  • The </h1> tag is the end tag.

Exercise:

<p>This is a paragraph

Self-Closing Tags

Some tags don’t need an end tag because they don’t have any content. These are called self-closing tags.

For example:

<br> (line break)
<img src="image.jpg" alt="Description"> (image tag)

Self-closing tags are often used for elements that don’t contain any content, like images or line breaks.


Understanding the Structure

Here’s a simple example that uses tags and elements:

<html>
  <head>
    <title>My First Page</title>
  </head>
  <body>
    <h1>Welcome to My Website</h1>
    <p>This is my first webpage.</p>
  </body>
</html>

In this example:

  • The <html> element contains the entire document.
  • The <head> element contains metadata (like the page title).
  • The <body> element contains the visible content of the webpage.

Key Takeaway

Every webpage is built with HTML tags and elements. By understanding these building blocks, you’ll be able to create and structure content for any webpage!

HTML Structure Trick

Did you know that many code editors have built-in shortcuts to save you time? Here's a quick trick:

If you're working in a code editor like VS Code, you can type ! in an empty HTML file and press Enter.


This will automatically generate the basic HTML structure for your file:

<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document</title>
  </head>
  <body>
    <!-- Your content goes here -->
  </body>
</html>

This trick is a great alternative until we dive deeper into HTML structure in a future lesson.

Give it a try and see how much time it saves you!
