Introduction to Java

Java is a versatile and powerful programming language widely used for web development, mobile applications, and enterprise systems. Known for its "write once, run anywhere" capability, Java is an excellent choice for beginners and professionals alike.

Why Learn Java?

  • Widely used in industries for building robust applications.
  • Object-oriented programming language, which makes code reusable and modular.
  • Strong community support and a plethora of libraries.

Getting Started with Java

To start coding in Java, you need:

  • An IDE or code editor to input and output your code. Learn how to set one up.
  • The Java Development Kit (JDK). Download it from the official website.
  • A basic understanding of programming concepts (optional but helpful for beginners).

Example: Hello World in Java

public class Main {
  public static void main(String[] args) {
    System.out.println("Hello, World!");
  }
}

Variables and Data Types

Variables are containers for storing data. Data types define the kind of data a variable can hold. Java is a statically-typed language, meaning each variable must be declared with a data type.

Common Data Types

  • int: Stores integers (e.g., 1, 2, 3).
  • double: Stores decimal numbers (e.g., 3.14, 2.718).
  • char: Stores single characters (e.g., 'A', 'B').
  • String: Stores sequences of characters (e.g., "Hello").
  • boolean: Stores true or false values.

Declaring Variables

To declare a variable in Java, specify its data type followed by the variable name:

int myNumber;
double myDecimal;
String myText;

Assigning Values to Variables

To assign a value to a variable, you can declare it and add its value on the end, or if already declared, just write its name and value:

myNumber = 10;
double Pi = 3.14;
myText = "Hello, Java!";

Variable Scope

The scope of a variable determines where it can be accessed:

  • Local Variables: Declared inside methods or blocks and only accessible within that scope.
  • Instance Variables: Declared inside a class but outside methods. They belong to an object of the class.
  • Static Variables: Declared using the static keyword and shared across all instances of the class.

Example

public class VariablesExample {
  public static void main(String[] args) {
    int age = 25;
    String name = "Jessica";
    boolean isStudent = true;
    System.out.println(name + " is " + age + " years old.");
  }
}

Decision-Making in Java

Decision-making statements allow your program to execute certain code blocks based on specific conditions. This is crucial for creating dynamic and logical applications.

1. The `if` Statement

The `if` statement executes a block of code only if a specified condition is true.

int x = 10;
if (x > 0) {
  System.out.println("x is positive");
}

In this example, the condition `x > 0` is true, so the program prints "x is positive."

2. The `else` Statement

The `else` statement executes a block of code if the condition in the `if` statement is false.

int x = -5;
if (x > 0) {
  System.out.println("x is positive");
} else {
  System.out.println("x is not positive");
}

3. The `else if` Statement

The `else if` statement allows you to test multiple conditions in sequence.

int x = 0;
if (x > 0) {
  System.out.println("x is positive");
} else if (x < 0) {
  System.out.println("x is negative");
} else {
  System.out.println("x is zero");
}

4. The `switch` Statement

The `switch` statement evaluates an expression and executes code based on the matching case.

int day = 3;
switch (day) {
  case 1:
    System.out.println("Monday");
    break;
  case 2:
    System.out.println("Tuesday");
    break;
  case 3:
    System.out.println("Wednesday");
    break;
  default:
    System.out.println("Other day");
}

In this example, the value of `day` matches `case 3`, so "Wednesday" is printed.

Loops in Java

Loops allow you to execute a block of code repeatedly as long as a condition is true. Java provides several types of loops to handle different scenarios.

1. For Loop

The `for` loop is used when the number of iterations is known in advance. It has three parts: initialization, condition, and increment/decrement.

for (int i = 0; i < 5; i++) {
  System.out.println("Count: " + i);
}

In this example:

  • Initialization: int i = 0; starts the counter at 0.
  • Condition: i < 5; repeats the loop as long as this condition is true.
  • Increment: i++; increases the counter (value of i) by 1 after each iteration.

2. While Loop

The `while` loop executes as long as its condition is true. It is often used when the number of iterations is not known in advance.

int i = 0;
while (i < 5) {
  System.out.println("Count: " + i);
  i++;
}

The loop starts with i = 0 and checks the condition i < 5. It repeats until the condition becomes false.

3. Do-While Loop

The `do-while` loop executes the block of code at least once, even if the condition is false initially.

int i = 0;
do {
  System.out.println("Count: " + i);
  i++;
} while (i < 5);

In this example, the loop prints "Count: 0" before checking the condition i < 5.

4. Nested Loops

A loop inside another loop is called a nested loop. This is useful for working with grids or tables.
The inner loop is completed one time for each outer loop.

for (int i = 1; i <= 3; i++) {
  for (int j = 1; j <= 3; j++) {
    System.out.print("(" + i + ", " + j + ") ");
}
  System.out.println();
}

This example creates a 3x3 grid, where the outer loop controls rows and the inner loop controls columns.

Best Practices for Using Loops

  • Avoid infinite loops by ensuring your condition will eventually become false.
  • Use meaningful variable names to make your code readable.
  • Keep your loop body concise to improve performance and clarity.

Introduction to Methods

Methods in Java are reusable blocks of code designed to perform specific tasks. They allow you to write cleaner, more modular, and reusable code, improving both readability and maintainability.

1. What are Methods?

A method in Java is a block of code that is executed when it is called. It can accept input parameters, perform operations, and optionally return a value. Think of methods as tools or instructions for performing a task.

2. Why Use Methods?

  • Code Reusability: Write once, use multiple times.
  • Modularity: Break large programs into smaller, manageable pieces.
  • Ease of Maintenance: Debug and update your program more easily.

3. Structure of a Method

public returnType methodName(parameters) {
    // Code to execute
    return value; // Optional
}

Key Components:

  • Access Modifier: Defines who can access the method (e.g., public, private).
  • Return Type: Specifies what the method will return (use void if it doesn't return anything).
  • Method Name: Describes what the method does.
  • Parameters: Optional inputs that the method accepts.
  • Method Body: Contains the actual code that executes.

4. Example: A Simple Method

public class Greeting {
    public void sayHello() {
        System.out.println("Hello, World!");
    }
}

To call this method:

public class Main {
    public static void main(String[] args) {
        Greeting greeting = new Greeting(); // Create object
        greeting.sayHello(); // Call method
    }
}

Parameter Passing and Return Values in Java

Parameters and return values are essential for methods to communicate with the rest of your program. They allow methods to accept inputs and send results back.

1. What are Parameters?

Parameters are inputs that a method accepts when it is called. They allow you to pass data to the method for processing. Parameters are defined inside the parentheses of a method declaration.

public void greet(String name) {
    System.out.println("Hello, " + name);
}

In this example, String name is a parameter. When the method is called, the caller must provide a value for name.

2. Types of Parameters

Java methods can accept different types of parameters:

  • Primitive Types: e.g., int, double, boolean.
  • Object Types: e.g., String, ArrayList, or custom objects.

3. Return Values

A method can send data back to the caller using a return value. The return type is specified in the method declaration. If the method does not return anything, use void.

public int add(int a, int b) {
    return a + b;
}

In this example, the method returns the sum of a and b. The keyword return is used to send the result back to the caller.

4. Example: Method with Parameters and a Return Value

public class MathOperations {
    public int multiply(int a, int b) {
        return a * b;
    }
}

public class Main {
    public static void main(String[] args) {
        MathOperations math = new MathOperations();
        int result = math.multiply(5, 10);
        System.out.println("The result is: " + result);
    }
}

In this example:

  • The method multiply accepts two integers, a and b.
  • It calculates their product and returns the result.
  • The result is printed in the main method.

Method Overloading in Java

Method overloading allows a class to have multiple methods with the same name but different parameters. It enables you to define methods that perform similar tasks but with different inputs.

1. What is Method Overloading?

In Java, method overloading means creating multiple methods with the same name in the same class but differing in:

  • The number of parameters.
  • The types of parameters.
  • The order of parameters.

2. Why Use Method Overloading?

Overloading improves code readability and reduces duplication by allowing you to handle different input scenarios with a single method name. It is particularly useful when:

  • Methods perform the same operation but require different types or numbers of inputs.
  • You want to provide default behaviors by overloading methods without parameters.

3. Rules for Method Overloading

  • Methods must have the same name.
  • Methods must differ in their parameter list (number, type, or order).
  • Return type alone cannot be used to distinguish overloaded methods.

4. Example: Method Overloading

public class Calculator {
    public int add(int a, int b) {
        return a + b;
    }

    public double add(double a, double b) {
        return a + b;
    }

    public int add(int a, int b, int c) {
        return a + b + c;
    }
}

public class Main {
    public static void main(String[] args) {
        Calculator calc = new Calculator();
        System.out.println("Sum (int): " + calc.add(5, 10));
        System.out.println("Sum (double): " + calc.add(5.5, 10.5));
        System.out.println("Sum (3 int): " + calc.add(1, 2, 3));
    }
}

In this example:

  • The add method is overloaded to accept different numbers and types of parameters.
  • The correct version of the method is called based on the arguments provided.

Static and Instance Methods in Java

In Java, methods are categorized as either static or instance methods. Understanding the difference is crucial for determining how methods are accessed and how they interact with the data in your program.

1. What are Static Methods?

Static methods belong to the class rather than a specific instance of the class. They can be called directly using the class name, without creating an object of the class.

public class MathUtils {
    public static int square(int number) {
        return number * number;
    }
}

public class Main {
    public static void main(String[] args) {
        int result = MathUtils.square(5); // Call static method
        System.out.println("Square: " + result);
    }
}

In this example, the square method is static and called directly using the class name MathUtils.

2. What are Instance Methods?

Instance methods belong to an object of the class. They can access and modify instance variables and require creating an object of the class to be called.

public class Person {
    private String name;

    public Person(String name) {
        this.name = name;
    }

    public void introduce() {
        System.out.println("Hi, my name is " + name);
    }
}

public class Main {
    public static void main(String[] args) {
        Person person = new Person("Jessica"); // Create object
        person.introduce(); // Call instance method
    }
}

In this example, the introduce method is an instance method, so it requires creating an object of the Person class.

3. Key Differences

Feature Static Methods Instance Methods
Belongs To: The class An object
Access: Called using the class name Called using an object
Access to Instance Variables: No Yes
Usage: For utility or helper methods For object-specific behavior

Best Practices and Use Cases for Methods

Writing methods is a core part of Java programming. To ensure your methods are effective and maintainable, it's important to follow certain best practices and understand common use cases.

1. Best Practices for Writing Methods

  • Use Descriptive Names: Method names should clearly describe what the method does. For example, calculateAverage is better than calc.
  • Keep Methods Short: Each method should perform a single task. If a method grows too long, consider splitting it into smaller methods.
  • Avoid Side Effects: Methods should only do what they are intended to. Avoid modifying global variables or producing unexpected results.
  • Use Parameters and Return Values Effectively: Pass data as parameters and return results instead of relying on external state.
  • Comment Complex Logic: Add comments to explain complex sections of code or algorithms.

2. Common Use Cases for Methods

Methods can be applied in various ways to simplify your program. Here are some common use cases:

  • Utility Methods: General-purpose methods that can be reused, such as mathematical calculations or string manipulation.
  • Getters and Setters: Methods to access or modify private fields of a class.
  • Modular Code: Break large programs into smaller, manageable pieces using methods.
  • Event Handlers: Methods triggered by user actions, such as button clicks in GUI programs.

3. Example: Utility and Modular Methods

public class StringUtils {
    public static String capitalize(String str) {
        if (str == null || str.isEmpty()) return str;
        return str.substring(0, 1).toUpperCase() + str.substring(1);
    }
}

public class Main {
    public static void main(String[] args) {
        String input = "java";
        System.out.println(StringUtils.capitalize(input));
    }
}

In this example, the capitalize method is a reusable utility method to capitalize the first letter of a string.

OOP Concepts

Object-Oriented Programming (OOP) is a programming paradigm that organizes code around objects rather than functions or logic. These objects represent real-world entities, allowing for more intuitive and modular software design.

  • Modularity: Code is organized into reusable objects and classes.
  • Reusability: Classes can be reused across programs, reducing redundancy.
  • Scalability: OOP makes it easier to expand and maintain software.
  • Data Security: Encapsulation hides sensitive data and enforces controlled access.
  • Real-World Modeling: Objects closely mimic real-world entities, making software easier to design and understand.

Object-Oriented Programming (OOP) is a programming approach that revolves around **objects** and **classes**. It simplifies programming by structuring code into reusable and modular pieces. Let's explore the building blocks of OOP: objects and classes.

1. What is a Class?

A **class** is like a blueprint or template for creating objects. It defines the structure and behavior of the objects. Think of it as a plan for a house—it describes what the house will look like and what it can do, but it isn't the house itself.

public class Car {
    String brand; // Field
    String model; // Field

    public void drive() { // Method
        System.out.println("The car is driving.");
    }
}

In this example, the `Car` class describes a car's **attributes** (fields like `brand` and `model`) and its **behavior** (method `drive()`).

2. What is an Object?

An **object** is an instance of a class. If a class is the blueprint, an object is the actual house built from that blueprint. Objects represent specific examples of the class with their own unique data.

public class Main {
    public static void main(String[] args) {
        Car myCar = new Car(); // Create an object
        myCar.brand = "Toyota"; // Set field
        myCar.model = "Corolla"; // Set field
        myCar.drive(); // Call method
    }
}

Here, `myCar` is an object created from the `Car` class. It has specific values for its fields (`brand` = "Toyota", `model` = "Corolla") and can perform the behavior defined in the class (e.g., calling `drive()`).

3. Key Differences Between Classes and Objects

Aspect Class Object
Definition A blueprint or template. An instance of a class.
Purpose Defines structure and behavior. Stores actual data and performs actions.
Example Car My car, your car

4. Why Use Classes and Objects?

Classes and objects make your code:

  • Reusable: Write a class once, create multiple objects from it.
  • Modular: Organize code into logical, manageable pieces.
  • Scalable: Build complex systems using interconnected objects.

Objects and Classes

Objects and classes are at the heart of Java programming. A class is a blueprint that defines the structure and behavior of an object, while an object is an instance of a class, representing a specific entity in the real world.

What is a Class?

A class serves as a blueprint or template that defines the properties (fields) and behaviors (methods) of an object.

public class Car {
    String brand; // Field
    String color; // Field

    public void drive() { // Method
        System.out.println("The car is driving.");
    }
}

Here, the class Car defines two fields (brand and color) and one method (drive()).

Creating Classes

A class can be created in the same file as the main program or in a separate file. Each approach has its own use case and rules.

Creating a Class in the Same File

You can define additional classes in the same file as your main class, but:

  • Only one class in the file can be declared public.
  • The file name must match the name of the public class.
public class Main {
    public static void main(String[] args) {
        Car myCar = new Car();
        myCar.brand = "Tesla";
        myCar.drive();
    }
}

class Car {
    String brand;
    public void drive() {
        System.out.println("Driving the car.");
    }
}

Creating a Class in a Separate File

To improve readability and organization, you can define each class in its own file. This is the preferred approach for larger programs.

  • Each file contains one class.
  • The file name must match the class name.
// Main.java
public class Main {
    public static void main(String[] args) {
        Car myCar = new Car();
        myCar.brand = "Tesla";
        myCar.drive();
    }
}

// Car.java
public class Car {
    String brand;
    public void drive() {
        System.out.println("Driving the car.");
    }
}

Key Differences

Aspect Same File Separate File
Organization All classes in one file, harder to navigate. Each class in its own file, easier to locate/edit.
Code Reusability Limited reusability outside the file. Classes can be reused across projects.
Complexity Good for small programs or examples. Preferred for larger, modular programs.

What is an Object?

An object is a specific instance of a class. It has its own unique values for the fields defined in the class.

public class Main {
    public static void main(String[] args) {
        Car myCar = new Car(); // Create object
        myCar.brand = "Tesla"; // Assign field
        myCar.color = "Red"; // Assign field
        myCar.drive(); // Call method
    }
}

In this example, the object myCar is an instance of the Car class. It has specific values for its fields (brand = "Tesla", color = "Red").

How to Create Objects

Objects are created using the new keyword followed by the class constructor.

ClassName objectName = new ClassName();

For example:

Car myCar = new Car();

Class Fields

Class fields (also called attributes or properties) store data for each object created from a class. Understanding how to initialize, update, and manage these fields is essential for working with objects effectively.

1. Initializing Fields

Fields in a class can be initialized in several ways:

  • Default Initialization: If you don't initialize a field, Java assigns a default value (e.g., 0 for integers, null for objects).
  • Explicit Initialization: Fields can be given a value directly when declared.
  • Constructor Initialization: Fields can also be initialized via constructor methods, providing flexibility to set values during object creation.
public class Car {
    String brand = "Unknown"; // Explicit initialization
    int year; // Default initialization

    public Car(String brand, int year) { // Constructor method initialization
        this.brand = brand;
        this.year = year;
    }
}

Note: Constructors and "this." statements will be fully covered in a future topic.

2. Updating and Overriding Field Values

Field values can be updated after the object is created by directly accessing them (if they're public) or through setter methods (if encapsulated).

Car myCar = new Car("Tesla", 2020);
myCar.year = 2022; // Direct update
System.out.println(myCar.year); // Outputs: 2022

Using setter methods ensures controlled updates, especially when additional logic is required.

public void setYear(int year) {
    if (year > 1886) { // Ensure a valid year
        this.year = year;
    }
}

Note: Setter and Getter methods will be covered fully in a future topic.

3. Using the final Keyword

The final keyword in Java is used to declare fields that cannot be changed after they are initialized. This ensures the value of a final field remains constant for the lifetime of the object.

3.1 When to Use final

Use final when:

  • You want to create constants (e.g., PI, fixed IDs).
  • You need to ensure a field cannot be modified accidentally after initialization.

3.2 Rules for final Fields

Fields declared final must be assigned a value:

  • At the point of declaration:
    • public class Circle {
        final double PI = 3.14159; // Value set at declaration
    }
  • Or in the constructor:
    • public class Car {
        final String brand;

        public Car(String brand) {
            this.brand = brand; // Value set in constructor
        }
    }

Once a value is assigned, it cannot be changed. Attempting to reassign a final field results in a compilation error.

4. Changing Field Values for a Single Object

Each object of a class has its own copy of fields. Updating a field in one object does not affect other objects.

Car car1 = new Car("Tesla", 2020);
Car car2 = new Car("Ford", 2019);

car1.year = 2022; // Only car1's year changes
System.out.println(car1.year); // Outputs: 2022
System.out.println(car2.year); // Outputs: 2019

Constructors in Java

A constructor is a special method used to initialize objects in Java. It is automatically called when an object is created, allowing you to set up initial values for fields or perform setup tasks.

1. What is a Constructor?

A constructor is a type of method, but with some key differences:

  • It has the same name as the class.
  • It does not have any return type (not even void).
  • It is automatically called when you create an object using the new keyword.
public class Car {
    String brand;
    String color;

    public Car() { // Constructor
        brand = "BMW";
        color = "Unknown";
    }
}

Car myCar = new Car(); // Constructor is called here

In this example, the constructor sets default values for the brand and color fields when a Car object is created.

2. Types of Constructors

2.1 Default Constructor

A default constructor is one that does not take any parameters. It is often used to assign default values to fields.

public class Person {
    String name;
    int age;

    public Person() {
        name = "Unknown";
        age = 0;
    }
}

When you create an object of the Person class, the default constructor sets name to "Unknown" and age to 0.

2.2 Parameterized Constructor

A parameterized constructor uses "this." statements to take arguments to set custom values for the fields of an object during creation.

public class Person {
    String name;
    int age;

    public Person(String name, int age) {
        this.name = name;
        this.age = age;
    }
}

Person person1 = new Person("Jessica", 25); // Constructor with parameters

In this example, the fields name and age are set to "Jessica" and 25 when the object is created.

3. Overloading Constructors

Like methods, constructors can be overloaded. This means you can define multiple constructors with different parameter lists to provide flexibility in object creation.

public class Car {
    String brand;
    String color;

    public Car() {
        brand = "Unknown";
        color = "Black";
    }

    public Car(String brand, String color) {
        this.brand = brand;
        this.color = color;
    }
}

Here, you can create a Car object with default values or provide specific values for brand and color.

Class Methods in Java

A class method defines the behavior or actions of an object. Methods allow you to organize your code into reusable blocks that perform specific tasks.

What is a Class Method?

A class method is a function defined inside a class. It is used to perform operations on the fields of an object or provide functionality specific to the class.

public class Car {
    String brand;
    String color;

    public void drive() { // Method
        System.out.println("The " + color + " " + brand + " is driving.");
    }
}

In this example, the drive() method performs an action using the object's fields.

How to Define a Method

A method is defined using the following syntax:

returnType methodName(parameters) {
    // Method body
}

- returnType: The type of value the method returns (use void if it doesn't return anything).
- methodName: The name of the method.
- parameters: Inputs to the method (optional).

Example: Adding a Method to a Class

Here’s an example of a class with a method:

public class Person {
    String name;
    int age;

    public void introduce() {
        System.out.println("Hi, my name is " + name + " and I am " + age + " years old.");
    }
}

In this example, the introduce() method prints the object's name and age.

Method Parameters

Methods can take parameters, allowing you to pass values when calling them. These parameters act as inputs for the method.

public void setDetails(String newName, int newAge) {
    name = newName;
    age = newAge;
}

To call this method, pass the required arguments:

Person person1 = new Person();
person1.setDetails("Jessica", 25);

The method setDetails(), has 2 parameters and therefore takes in 2 arguments, "newName" and "newAge". These are then inputted into the variables name and age.

public int getAge() {

In this example, the getAge() method returns the value of the age field.

Public vs Static in Java

In Java, the public and static keywords define how methods or variables can be accessed and used. Understanding their differences is essential for designing efficient and logical code.

Public

The public keyword determines the access level of a class, method, or variable. Anything declared as public can be accessed from any other class in your program.

  • Usage: Use public when you need to make something accessible across different parts of your program.
  • Example:
public class Person {
    public String name; // Accessible from other classes

    public void greet() {
        System.out.println("Hello, my name is " + name);
    }
}

Here, the name field and greet() method are accessible from any other class that creates a Person object.

Static

The static keyword makes a method or variable belong to the class itself rather than an instance (object) of the class. This means:

  • Static Methods: Can be called without creating an object of the class.
  • Static Variables: Shared among all instances of the class.
public class MathUtils {
    public static double add(double a, double b) {
        return a + b;
    }
}

double sum = MathUtils.add(5, 3); // No object creation needed

In this example, the add() method is called directly using the class name because it is static.

Public vs Static

Let’s compare public and static:

Aspect Public Static
Scope Defines access to classes, methods, or fields. Defines whether a method or field belongs to the class or an instance.
Access Can be accessed by all other classes. Can be accessed without creating an object of the class.
Example person.greet() MathUtils.add()

When to Use Public vs. Static

1. Use public When:

  • You want a method or variable to be accessible across your program.
  • The behavior or value is tied to individual objects, not the class.
public class Car {

    public void drive() {
        System.out.println("Driving the " + model);
    }
}

Car myCar = new Car();
myCar.model = "Tesla";
myCar.drive(); // Tied to the object myCar

2. Use static When:

  • The behavior or value doesn’t depend on any specific object.
  • You want to create utility methods or shared data.
public class Utils {
    public static String formatName(String name) {
        return name.toUpperCase();
    }
}

System.out.println(Utils.formatName("Jessica")); // Outputs: Jessica

Modifiers in Java

Modifiers in Java are keywords used to define the behavior, access, and properties of classes, attributes, methods, and variables. They allow you to control how different parts of your code interact.

Types of Modifiers

Modifiers can be categorized into:

  • Access Modifiers: Control the visibility of classes, methods, and attributes (e.g., public, private, protected, default).
  • Non-Access Modifiers: Provide additional properties to classes, methods, and attributes (e.g., final, static, abstract, transient, synchronized, volatile).

Final Modifier

The final modifier prevents modification:

  • For variables: A final variable cannot be changed after it’s assigned a value.
  • For methods: A final method cannot be modified once defined.
  • For classes: A final class cannot be changed.
// Final variable example
public class Constants {
    public final double PI = 3.14159;
    public final int MAX_USERS = 100;
}

// Final class example
public final class Utility {
    public static String formatText(String text) {
        return text.toUpperCase();
    }
}

// Final method example
public class BankAccount {
    private double balance = 0; // variable is not final, since value will hopefully be changed.

    public final void displayBalance() {
        System.out.println("The current balance is: $" + balance);
    }
}

Abstract Modifier

The abstract modifier is used to define classes or methods that act as templates or placeholders, without fully implementing functionality. They are designed to be extended later (but we won’t dive into subclassing yet).

Abstract Classes

An abstract class can contain:

  • Abstract methods: Methods that are declared but not implemented.
  • Concrete methods: Fully implemented methods, just like in regular classes.
public abstract class Shape {
    // Abstract method (no implementation)
    public abstract double calculateArea();

    // Concrete method (fully implemented)
    public void displayInfo() {
        System.out.println("This is a shape.");
    }
}

Although the calculateArea() method is defined as abstract, the displayInfo() method is implemented and can be used immediately.

Abstract Methods

Abstract methods are defined in abstract classes and provide a way to ensure certain functionality is implemented later (we’ll explore how this works in more detail in future sections). For now:

  • An abstract method does not have a body (i.e., it does not include a block of code).
  • It must be declared in an abstract class.
public abstract class Appliance {
    public abstract void turnOn(); // Abstract method

    public void describe() {
        System.out.println("This is an appliance.");
    }
}

In this example, turnOn() is abstract, while describe() is a concrete method that can already be used. We'll revisit the concept of implementing abstract methods when we cover subclasses in the future.

Public and Static

Other Modifiers

Here’s a quick overview of some other commonly used modifiers:

  • Transient: Used with fields to indicate they should not be serialized.
  • Synchronized: Used with methods to make them thread-safe by allowing only one thread at a time to access them.
  • Volatile: Indicates that a variable’s value may change unexpectedly due to multiple threads.

Encapsulation in Java

Encapsulation is the process of bundling data (fields) and methods (behavior) together in a class, while restricting direct access to the fields. Instead, access is provided through methods like getters and setters.

Why Use Encapsulation?

Encapsulation helps:

  • Protect Data: By making fields private, you prevent unintended modifications.
  • Control Access: Getters and setters allow you to validate or transform data when needed.
  • Improve Code Maintenance: Internal details of the class are hidden, so changes to fields don’t affect other parts of your program.

How to Implement Encapsulation

To achieve encapsulation, follow these steps:

  1. Make fields private.
  2. Provide public getter and setter methods to access and modify the fields.

Why do setters need parameters, but getters don’t?
- A setter needs a parameter because its purpose is to assign a new value to a field. Without a parameter, the setter wouldn’t know what value to set.
- A getter doesn’t need a parameter because it simply retrieves and returns the current value of a field.

public class Person {
    private String name; // Field is private
    private int age; // Field is private

    public String getName() { // Getter
        return name;
    }

    public void setName(String name) { // Setter
        this.name = name;
    }
}

The this Keyword

The this keyword refers to the current object and is often used in setter methods to avoid naming conflicts. Without this, you might need separate variable names to avoid ambiguity.

public void setName(String name) {
    this.name = name; // Resolves ambiguity
}

Without this, you would need to use a different parameter name:

public void setName(String newName) {
    name = newName;
}

When to Use Getter and Setter Methods

Getter and setter methods are used to control how fields are accessed or modified:

  • Getters: Use to retrieve the value of a field. Useful when you want to expose data but keep it read-only.
  • Setters: Use to update the value of a field. You can include validation to ensure data integrity.
public void setAge(int age) {
    if (age > 0) { // Validation
        this.age = age; // Setter statement
    } else {
        System.out.println("Age must be positive!");
    }
}

public int getAge() {
    return age;
}

Benefits of Encapsulation

Encapsulation helps to create secure and maintainable code. For example, by adding validation in setter methods, you can ensure your objects remain in a valid state:

Person person = new Person();
person.setAge(-5); // Prints: Age must be positive!
person.setAge(25); // Updates the age successfully
System.out.println(person.getAge()); // Prints: 25

Introduction to Collections

Collections in Java provide a framework for managing groups of objects. They simplify working with data by offering built-in methods for storing, retrieving, and manipulating collections of elements. Whether you're working with a list of names, a set of unique IDs, or a mapping of keys to values, collections offer a versatile and efficient solution.

Why Use Collections?

Collections are powerful because they:

  • Collection: The root interface for most collection types.

  • Set: Extends Collection, representing a collection with unique elements.
  • Map: Represents a mapping of unique keys to values.

ArrayList in Java

The ArrayList class in Java is a part of the java.util package and represents a resizable array. Unlike traditional arrays, an ArrayList can grow or shrink dynamically as elements are added or removed.

Key Features of ArrayList

  • Dynamic Resizing: Automatically adjusts its size when elements are added or removed.
  • Index-Based Access: Allows you to retrieve elements by their index, similar to arrays.
  • Allows Duplicates: An ArrayList can contain duplicate elements.
  • Maintains Order: Elements are stored in the order they were added.

How to Use an ArrayList

To use an ArrayList, you must import the java.util.ArrayList class. Here's an example of creating and using an ArrayList:

import java.util.ArrayList; // Importing ArrayList

public class Main {
    public static void main(String[] args) {
        // Create an ArrayList of Strings
        ArrayList<String> names = new ArrayList<>();

        // Add elements to the ArrayList
        names.add("James");
        names.add("Steve");
        names.add("Andrew");

        // Print the ArrayList
        System.out.println(names); // Output: [James, Steve, Andrew]

        // Remove an element
        names.remove("Steve");
        System.out.println(names); // Output: [James, Andrew]
    }
}

Common Methods in ArrayList

  • add(element): Adds an element to the list.
  • remove(element): Removes the first occurrence of the specified element.
  • get(index): Retrieves the element at the specified index.
  • size(): Returns the number of elements in the list.
  • contains(element): Checks if the list contains the specified element.
  • clear(): Removes all elements from the list.

Advantages of ArrayList

  • Provides dynamic resizing without manual memory management.
  • Offers built-in methods for common operations like sorting, searching, and iteration.
  • Maintains the insertion order of elements.

Limitations of ArrayList

  • Slower performance compared to arrays for primitive types due to autoboxing and unboxing.
  • Not synchronized, meaning it is not thread-safe for concurrent operations.

LinkedList in Java

The LinkedList class in Java is a part of the java.util package. Unlike ArrayList, which uses a dynamic array to store its elements, a LinkedList uses a doubly linked list. Each element (or node) in a LinkedList contains both the data and a reference to the next and previous elements in the sequence.

Key Features of LinkedList

  • Dynamic Memory: Like ArrayList, it can grow or shrink dynamically as elements are added or removed.
  • Efficient Insertions and Deletions: Adding or removing elements from the middle of the list is faster compared to ArrayList.
  • Maintains Order: Elements are stored in the order they were added.
  • Can Act as a Queue: Implements both List and Deque interfaces, making it useful for queue and stack operations.

How to Use a LinkedList

To use a LinkedList, you must import the java.util.LinkedList class. Here's an example of creating and using a LinkedList:

import java.util.LinkedList;

public class Main {
    public static void main(String[] args) {
        // Create a LinkedList of Strings
        LinkedList<String> cities = new LinkedList<>();

        // Add elements to the LinkedList
        cities.add("Tokyo");
        cities.add("Oslo");
        cities.add("Nairobi");

        // Print the LinkedList
        System.out.println(cities); // Output: [Tokyo, Oslo, Nairobi]

        // Remove an element
        cities.remove("Oslo");
        System.out.println(cities); // Output: [Tokyo, Nairobi]

        // Add an element at the beginning
        cities.addFirst("Sydney");
        System.out.println(cities); // Output: [Sydney, Tokyo, Nairobi]
    }
}

Common Methods in LinkedList

  • add(element): Adds an element to the end of the list.
  • addFirst(element): Adds an element to the beginning of the list.
  • addLast(element): Adds an element to the end of the list.
  • remove(element): Removes the occurrence of the specified element.
  • removeFirst(element): Removes the first occurrence of the specified element.
  • removeLast(element): Removes the last occurrence of the specified element.
  • get(index): Retrieves the element at the specified index.
  • size(): Returns the number of elements in the list.

When to Use LinkedList

LinkedList is preferred over ArrayList when:

  • You need to frequently insert or remove elements in the middle of the list.
  • Memory reallocation costs of resizing an array in ArrayList are a concern.

However, for scenarios where you need fast random access to elements, ArrayList is usually a better choice.

Sets in Java

A Set in Java is a collection that does not allow duplicate elements. It is part of the java.util package and comes in multiple implementations, such as HashSet, LinkedHashSet, and TreeSet, each with unique characteristics.

Key Features of Set

  • No Duplicates: Ensures that each element in the collection is unique.
  • Unordered or Ordered: The order of elements depends on the specific implementation.
  • Efficient Membership Testing: Provides efficient methods to check if an element exists in the set.

Common Implementations of Set

  • HashSet: Unordered and allows null values. Backed by a hash table for efficient operations.
  • LinkedHashSet: Maintains the order of elements based on their insertion.
  • TreeSet: Orders elements in their natural order or by a custom comparator.

How to Use a Set

Here's an example of using a HashSet:

import java.util.HashSet;

public class Main {
    public static void main(String[] args) {
        // Create a HashSet of Strings
        HashSet<String> fruits = new HashSet<>();

        // Add elements to the Set
        fruits.add("Apple");
        fruits.add("Banana");
        fruits.add("Cherry");
        fruits.add("Apple"); // Duplicate element, ignored

        // Print the Set
        System.out.println(fruits); // Output: [Banana, Cherry, Apple]

        // Check if an element exists
        System.out.println(fruits.contains("Apple")); // Output: true

        // Remove an element
        fruits.remove("Banana");
        System.out.println(fruits); // Output: [Cherry, Apple]
    }
}

Advantages of Using a Set

  • Ensures unique elements without duplicates.
  • Efficient for tasks like membership testing.
  • Flexible with multiple implementations to suit different needs.

Limitations of Set

  • Does not allow duplicate elements, which may be restrictive in some cases.
  • Random access (like retrieving an element by index) is not possible.

When to Use a Set

Use a Set when:

  • You need to store unique elements without duplicates.
  • You need efficient membership testing.
  • The order of elements is not critical, or you can choose an ordered implementation like LinkedHashSet or TreeSet.

Maps in Java

A Map in Java is a collection that stores data in key-value pairs. Each key is unique, and each key maps to exactly one value. Maps are part of the java.util package and are widely used for fast lookups and associations between data.

Key Features of Map

  • Unique Keys: Ensures that no two keys in a map are the same.
  • Key-Value Association: Each key maps to a specific value, making it efficient for lookups.
  • Various Implementations: Different types of maps provide different characteristics (e.g., order, performance).

Understanding Keys and Values

A Map stores data as a pair: a key and a value. Here's a breakdown:

  • Key: A unique identifier used to look up values in the map. Keys must be unique within a map.
  • Value: The data associated with a key. Values can be duplicated across different keys.

For example, consider this line of code:

ages.put("Anna", 25);

Here:

  • Key: "Anna" (a unique identifier).
  • Value: 25 (data associated with the key).

You can retrieve the value 25 by using its key "Anna", like this:

System.out.println(ages.get("Anna")); // Output: 25

Common Implementations of Map

  • HashMap: Unordered, allows one null key, and multiple null values. Provides constant-time performance for basic operations.
  • LinkedHashMap: Maintains the insertion order of keys.
  • TreeMap: Orders keys based on their natural ordering or a custom comparator.

How to Use a Map

Here's an example of using a HashMap:

import java.util.HashMap;

public class Main {
    public static void main(String[] args) {
        // Create a HashMap
        HashMap<String, Integer> ages = new HashMap<>();

        // Add key-value pairs to the Map
        ages.put("Anna", 25);
        ages.put("Ben", 30);
        ages.put("Clara", 22);

        // Retrieve a value using a key
        System.out.println("Anna's age: " + ages.get("Anna"));

        // Check if a key exists
        System.out.println("Contains Ben? " + ages.containsKey("Ben"));

        // Remove a key-value pair
        ages.remove("Clara");
        System.out.println(ages); // Output: {Anna=25, Ben=30}
    }
}

Common Methods in Map

  • put(key, value): Adds a key-value pair to the map.
  • get(key): Retrieves the value associated with the specified key.
  • remove(key): Removes the key-value pair associated with the specified key.
  • containsKey(key): Checks if the map contains the specified key.
  • containsValue(value): Checks if the map contains the specified value.
  • size(): Returns the number of key-value pairs in the map.

When to Use a Map

Use a Map when:

  • You need to store data with a clear association between keys and values.
  • Fast lookups are required.
  • Duplicate keys are not allowed, but duplicate values are acceptable.

Autoboxing and Unboxing in Java

In Java, Autoboxing and Unboxing are mechanisms that allow seamless conversion between primitive data types (e.g., int, double) and their corresponding wrapper classes (e.g., Integer, Double). These features are particularly useful when working with collections, which can only store objects.

What is Autoboxing?

Autoboxing is the automatic conversion of a primitive type into its corresponding wrapper class object. For example, when you add a primitive value to a collection, Java automatically "boxes" the primitive into its wrapper class.

import java.util.ArrayList;

public class Main {
    public static void main(String[] args) {
        // Autoboxing example
        ArrayList<Integer> numbers = new ArrayList<>();
        numbers.add(10); // int is autoboxed to Integer
        numbers.add(20);

        System.out.println(numbers); // Output: [10, 20]
    }
}

What is Unboxing?

Unboxing is the reverse process where a wrapper class object is automatically converted back into its corresponding primitive type. For example, when retrieving an element from a collection, Java automatically "unboxes" the wrapper object into a primitive.

public class Main {
    public static void main(String[] args) {
        ArrayList<Integer> numbers = new ArrayList<>();
        numbers.add(10);
        numbers.add(20);

        // Unboxing example
        int sum = numbers.get(0) + numbers.get(1); // Integer is unboxed to int
        System.out.println("Sum: " + sum); // Output: Sum: 30
    }
}

When to Use Autoboxing and Unboxing

These features are most commonly used in situations where primitives need to interact with objects, such as:

  • Storing primitive values in collections like ArrayList or HashMap.
  • Returning wrapper objects from methods that require primitives.
  • Performing operations on values retrieved from collections.

Performance Considerations

  • Autoboxing and unboxing introduce slight overhead because of the conversion process.
  • Frequent boxing and unboxing in performance-critical applications should be avoided.

Key Methods in Wrapper Classes

Wrapper classes provide utility methods for converting between primitives and their object equivalents:

  • Integer.parseInt(String s): Converts a string to an int.
  • Double.valueOf(String s): Converts a string to a Double object.
  • intValue(), doubleValue(), etc.: Extracts the primitive value from a wrapper object.

Inheritance in Java

Inheritance is a cornerstone of Object-Oriented Programming (OOP). It allows one class (the child or subclass) to inherit properties and behaviors (fields and methods) from another class (the parent or superclass). This promotes code reuse and helps in building hierarchies of related classes.

What is Inheritance?

Inheritance allows a class to acquire properties (fields) and behaviors (methods) of another class. The main class that provides its properties is called the parent (or superclass), and the class that inherits is called the child (or subclass).

class Animal {
  void eat() {
    System.out.println("This animal eats food.");
  }
}

class Dog extends Animal {
  void bark() {
    System.out.println("The dog barks.");
  }
}

public class Main {
  public static void main(String[] args) {
    Dog myDog = new Dog();
    myDog.eat(); // Inherited from Animal
    myDog.bark(); // Defined in Dog
  }
}

In this example:

  • Animal is the parent class with a method eat().
  • Dog is the child class that extends Animal and adds its own method bark().
  • The Dog object can access both eat() and bark().

Method Overriding

Inheritance also allows subclasses to provide a specific implementation for methods defined in the parent class. This is called method overriding.

class Animal {
  void sound() {
    System.out.println("This animal makes a sound.");
  }
}

class Dog extends Animal {
  @Override
  void sound() {
    System.out.println("The dog barks.");
  }
}

public class Main {
  public static void main(String[] args) {
    Animal myDog = new Dog();
    myDog.sound(); // Calls overridden method in Dog
  }
}

Here, the sound() method in Dog overrides the sound() method in Animal.

Benefits of Inheritance

  • Promotes code reuse by inheriting properties and methods from a parent class.
  • Allows for a hierarchical organization of classes, simplifying code readability and maintenance.
  • Supports method overriding, enabling flexibility and customized behavior in subclasses.

Polymorphism in Java

Polymorphism allows objects to take many forms. It is one of the key principles of Object-Oriented Programming (OOP) and provides flexibility and reusability in code. Polymorphism in Java can be achieved through method overriding and method overloading.

What is Polymorphism?

Polymorphism enables a single interface to be used for different types of objects. It allows the same action to behave differently depending on the context.

  • Compile-time Polymorphism: Achieved through method overloading.
  • Runtime Polymorphism: Achieved through method overriding.

1. Method Overloading

Method overloading occurs when multiple methods in the same class share the same name but have different parameter lists. It allows methods to perform similar tasks but with different inputs.

class Calculator {
  // Overloaded method
  int add(int a, int b) {
    return a + b;
  }
  double add(double a, double b) {
    return a + b;
  }
}

public class Main {
  public static void main(String[] args) {
    Calculator calc = new Calculator();
    System.out.println(calc.add(5, 10)); // Calls int version
    System.out.println(calc.add(3.5, 7.2)); // Calls double version
  }
}

2. Method Overriding

Method overriding occurs when a subclass provides a specific implementation for a method already defined in its superclass. It is a key feature of runtime polymorphism.

class Animal {
  void sound() {
    System.out.println("This animal makes a sound.");
  }
}

class Dog extends Animal {
  @Override
  void sound() {
    System.out.println("The dog barks.");
  }
}

public class Main {
  public static void main(String[] args) {
    Animal myDog = new Dog();
    myDog.sound(); // Outputs: "The dog barks."
  }
}

Benefits of Polymorphism

  • Simplifies code and improves readability.
  • Enhances reusability and flexibility.
  • Enables dynamic method invocation at runtime.
