JS Let and Var

In JavaScript, variables can be declared using var or let. Both allow you to store data, but they behave differently in certain situations.

Using var

The var keyword has been in JavaScript since the beginning. It allows you to declare a variable, which can be reassigned later.

Example:

var name = "Daniel";
console.log(name); // Output: Daniel

name = "Sophia";
console.log(name); // Output: Sophia

Since var allows reassignment, the value of name was changed from "Daniel" to "Sophia".

Using let

The let keyword was introduced in ES6 and is the preferred way to declare variables in modern JavaScript. Like var, it allows reassignment, but it is more predictable.

Example:

let age = 25;
console.log(age); // Output: 25

age = 30;
console.log(age); // Output: 30

Here, we declared a variable age using let, and we successfully changed its value later.

Key Differences Between var and let

While both var and let allow you to declare variables, there are some important differences.

  • Block Scope: let is block-scoped, meaning it only exists within the block (`{}`) it was declared in. var is function-scoped, meaning it exists throughout the entire function.
  • Redeclaration: A variable declared with var can be redeclared in the same scope. With let, redeclaring a variable in the same scope causes an error.

Example: Block Scope

function testVar() {
  if (true) {
    var x = 10;
  }
  console.log(x); // Output: 10 (x exists outside the block)
}

function testLet() {
  if (true) {
    let y = 20;
  }
  console.log(y); // Error: y is not defined outside the block
}

When using var, the variable leaks out of the block and is still accessible. With let, the variable is only available inside the block.

Example: Redeclaration

var a = 5;
var a = 10;
console.log(a); // Output: 10 (Redeclaration is allowed)

let b = 5;
let b = 10; // Error: Cannot redeclare a let variable

As you can see, var allows the same variable to be redeclared, while let does not.

Which One Should You Use?

In modern JavaScript, let is the preferred choice because:

  • It helps prevent unintended variable redeclarations.
  • It has block scope, making code more predictable.
  • It avoids many issues that arise when using var in large applications.
